# Composer FFET Derivation Rules v2

## Rule

The FFET count is not preselected. Candidates are derived from source claims, existing substrate touchpoints, exact-file ownership boundaries, runtime/artifact MCRs, and validation ladders.

## Split conditions

Split when a candidate owns multiple architectural decisions, spans unrelated validation ladders, touches overlapping file sets, creates multiple runtime subsystems, mixes policy interpretation with implementation, or requires Codex to invent missing rules.

## Merge conditions

Merge only when a written `merge_rationale` proves inseparability and the merged candidate still has one coherent MCR and exact-file scope.
